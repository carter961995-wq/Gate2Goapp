# Service Blueprint

## Intake Flow
1. User submits case details
2. AI plan generated with official links
3. Consent and ownership confirmation logged
4. Case status set to "Intake"

## Verification Flow
- Optional proof-of-ownership uploaded
- Verify Apple ID access status
- Assign escalation if verification fails

## Recovery Flow
- If device powers on + passcode known: Quick Start transfer
- If Apple ID access missing: account recovery via iforgot.apple.com
- If hardware damage: authorized repair routing
- If backup available: iCloud restore guidance

## Support Escalation
- Tiered response times by membership
- Human specialist handoff notes
- Customer updates every 24 hours

## Completion
- Confirm data restore success
- Provide backup setup checklist
- Close case with summary log

## Partner Handoff
- Only with explicit consent
- Share minimal case data
- Record timestamped handoff events
